# Architecture - Main

## Executive Summary

TimelessCalc is a web application for calculating and visualizing Path of Exile passive tree jewel configurations. It uses a canvas-based approach with Konva.js to render the passive skill tree and provides tools for finding optimal timeless jewel seeds.

## Architecture Pattern

**Component-Based Frontend Architecture**

- **Reactive Components**: Svelte components that automatically update when state changes
- **Store-Based State Management**: Centralized state using Svelte writable stores
- **Canvas Rendering**: Layered canvas approach for complex tree visualization
- **Data-Driven**: Static data files for tree and jewel information

## Data Architecture

### Data Sources
- **Static JSON Files**: Tree structure, translations, constants
- **Compressed JSONL Files**: Precomputed jewel statistics
- **External API**: PoE Watch for league information

### Data Flow
1. **Load Phase**: Static data loaded on application start
2. **Search Phase**: User selections filter and process data
3. **Display Phase**: Results rendered in UI and canvas

## API Design

### External APIs
- **PoE Watch Leagues**: Simple GET endpoint for league data
- **Local Data Loading**: Client-side loading of static assets

### No REST API
The application is frontend-only with no backend API endpoints.

## Component Overview

### UI Components (26 components organized in feature-based subfolders)
- **common/**: Shared components (Modal, Tooltip, Preloader, SidebarToggle)
- **search/**: Search interface and results components
- **selectors/**: Configuration form controls and selectors
- **modals/**: Modal dialog components
- **notifications/**: Notification and alert components
- **favorites/**: Favorites management interface
- **sidebar/**: Sidebar layout and navigation
- **debug/**: Development and debugging tools

### Canvas Components
- **Layers**: Background, nodes, connections, hit detection
- **Utilities**: Zoom, hover, click handling

## Development Workflow

### Prerequisites
- Node.js and npm

### Commands
- `npm run dev`: Development server
- `npm run build`: Production build
- `npm run check`: Type checking

## Deployment Architecture

### Build Output
- Static files generated by Vite
- Can be deployed to any static hosting service
- No server-side requirements

### No CI/CD
No automated deployment pipeline configured.

## Testing Strategy

### Current State
No automated testing framework implemented.

### Manual Testing
- Browser testing for UI interactions
- Canvas rendering verification
- Data loading validation

## Security Considerations

### External Data
- Fetches data from PoE Watch API
- Loads local static files
- No user data storage or authentication

### Content Security
- Client-side only application
- No server-side processing of user inputs

## Performance Characteristics

### Loading
- Lazy loading of jewel data
- Caching of API responses
- Compressed data files

### Rendering
- Canvas-based rendering for smooth interactions
- Reactive updates for UI responsiveness
- Debounced operations where applicable

## Future Considerations

### Scalability
- Static data approach limits dynamic updates
- Canvas performance may degrade with very large trees

### Maintainability
- TypeScript provides good maintainability
- Component-based architecture supports modular development
- Clear separation of concerns across directories

This architecture supports the core functionality of timeless jewel calculation and visualization while remaining simple and maintainable for a frontend-only application.
